# Роль: Куратор проекта

## Кто это

Куратор — ведущий студенческой команды в Проектном практикуме. Одна команда = один куратор. Семестровая роль.

## Кто может быть куратором

Сотрудник УрФУ, аспирант/магистрант с опытом в предметной области проекта, либо внешний индустриальный специалист по договору. Обязательное условие — возможность регулярного контакта с командой (минимум раз в 2 недели).

## Что обязательно, пока ты куратор

1. Проводить регулярные встречи с командой — минимум раз в 2 недели, обычно чаще.
2. Передавать предметный контекст и давать обратную связь по содержанию.
3. Принимать контрольные точки команды (согласовывать промежуточные результаты).
4. **Не входить в оценочную комиссию своей команды на защите.** Это жёсткое правило — разделение обязанностей.
5. Соблюдать дедлайны ПП, в том числе загрузку материалов в БЗ за 48 часов.

## Что в роль НЕ входит

- Работа за команду — тимлид и студенты управляют процессом сами.
- Финальная оценка в одиночку — ставит комиссия / модератор.
- Роль заказчика, даже если даёшь проект от индустрии (тогда эти роли разводятся явно).
- Административка: расписание, пересдачи, сборы заявок — зона организатора.

## С кем взаимодействуешь

| Контрагент | Характер |
|------------|----------|
| Тимлид | Регулярные встречи, уточнение задач, обратная связь |
| Команда | Групповые встречи, комментарии по работам |
| Партнёр | По необходимости — уточнение задачи |
| Модератор | При подготовке и проведении защиты |
| Организатор | Логистика, дедлайны, отчётность |
| Руководитель трека | Координация по треку (если куратор в треке) |

## Состояния роли

- **Назначен** — команда сформирована, привязка в ТИМПро есть, семестр ещё не начался.
- **Активен** — семестр идёт, встречи проводятся, контрольные точки принимаются.
- **Приостановлен** — болезнь, временная замена.
- **Завершён** — защита прошла, оценки выставлены, отчётность сдана.

## Специализация: Руководитель трека

Куратор с расширенными полномочиями (координация нескольких команд одного трека) и особыми условиями эффективного контракта. Карточка роли — появится отдельно (PP.R.NNN).

## Контрпримеры

- Куратор ≠ научный руководитель ВКР (другой процесс).
- Куратор ≠ преподаватель курса, в рамках которого идёт ПП.
- Куратор ≠ ментор в бизнес-акселерации (ПП — учебный формат).

## Связанные методы и правила

Появятся по мере добавления:
- Метод «Запуск семестра команды»
- Метод «Проведение контрольной точки»
- Метод «Подготовка к защите»
- Регламент дедлайнов ПП
